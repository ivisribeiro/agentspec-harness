01

The problem

Why prompt-driven spec workflows leak correctness

AgentSpec · build.md

Correctness lives in the model's head

  • Topological order is "Step 3" reasoning the LLM is asked to perform
  • "max 3 retries" is a line of prose the model can rationalise past
  • The build gate is a checkbox the LLM ticks on its own work

Every one is a place where, when the model misbehaves, nothing underneath catches it.

Before · AgentSpec

Prompt-driven

  • One LLM decides order, loops, gates
  • Retry cap & "done" live in prose
  • No schema at the worker handoff
  • Nothing a test can assert

After · Spindle

Gate-backed

  • Order, gates, state run in real code
  • Retry cap is data; "done" is a gate
  • spin complete --handoff validates first
  • Every step is an exit code a test checks

The keystone

Gates read disk + state, never the conversation

  • .spindle/run.json is a CLI-written ledger — never touched by a model
  • So gates are idempotent and crash-safe: re-run from the same files → same verdict
  • spin complete <id> --handoff validates the worker's JSON against its schema before marking done
.spindle/run.jsonschema · completed[] · retries{} · gates{}
.handoffs/<id>.jsonmachine-facing · Zod-validated
<ARTIFACT>.mdhuman-facing · read by people
03

The gates

spin gate <id> → exit 0 pass · exit 1 BLOCK

Eight invokable gates + G_HANDOFF

GateFires beforeBlocks if…
G_DEFINE/designsections missing, no AC-n ids, or define handoff invalid
G_DESIGN/buildno file-manifest table, or design handoff invalid
G_BUILD/shipany manifest file absent on disk, criteria-diff non-empty, or no report
G_SHIPpublishdefine.criteria minus build.passed is non-empty
G_KB_STRUCTUREKBdomain missing manifest / index / quick-reference / concepts
G_KB_COVERAGEKBa declared concept has no file, or too few test cases
G_ROUTER_COVERAGEgen-routerfrontmatter invalid, agent missing, or duplicated — no silent skip
G_REVIEW_BLOCK/review · /migratesurviving CRITICAL findings > 0

G_BUILD blocks, then advances — in code

the SDD cycle 
# build wrote only 1 of 2 manifest files
$ spin gate G_BUILD
{ "gate": "G_BUILD", "passed": false, "unmet": ["src/session.ts"] }   # exit 1 → STOP

# write the missing file, re-gate
$ spin gate G_BUILD
{ "gate": "G_BUILD", "passed": true }                                  # exit 0 → /ship

# the bounded loop is a counter in run-state, not prose
$ spin retry build --inc   # charges .spindle/run.json against build_retry_cap
$ spin retry build --ok    # exit 1 at the ceiling
04

Model routing

Haiku, Sonnet and Opus — the cheapest tier that can verifiably do it

Two hard rules, enforced in policy.test.ts

The verifier outranks the generator

  • An Opus adversary challenges a Sonnet build — the cheaper tier is never the final judge of a CRITICAL finding
  • Downgrade under --budget low only where a deterministic gate backstops the output
  • Critical kinds (adversary, architect, *-intent) never downgrade
HAIKUmechanical · gate-backstopped
SONNETanalysis · authoring
OPUSarchitecture · adversarial

Routing is pure, deterministic, and tested

spin route 
$ spin route adversary --budget low
{ "tier": "opus",   "reason": "adversarial — must outrank generator" }

$ spin route code-build --budget low
{ "tier": "haiku",  "reason": "downgraded — G_BUILD backstops it" }

$ spin route spec-authoring --budget low
{ "tier": "sonnet", "reason": "no gate backstop — stays sonnet" }
